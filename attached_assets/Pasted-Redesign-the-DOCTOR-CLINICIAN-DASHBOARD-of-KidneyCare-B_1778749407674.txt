Redesign the DOCTOR / CLINICIAN DASHBOARD of “KidneyCare BD” into a fully native‑feeling Progressive Web App (PWA) for mobile use. The existing dashboard handles patient lists, clinical alerts, patient timelines, telemedicine, e‑prescriptions, and GFR comparisons. Keep every existing feature, but repackage the entire doctor experience for a mobile‑first, bottom‑navigation workflow that a doctor can carry in their pocket during rounds in a Bangladeshi hospital or clinic.

1. PWA REQUIREMENTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- Fully installable PWA with manifest (name: “KidneyCare MD”, short_name: “KC MD”, display: standalone, theme_color #1A6B8A).
- Service worker using Workbox for offline caching of:
  - Patient list (last synced)
  - Recent alerts
  - Cached patient profiles (latest vitals + charts)
  - Static assets (icons, fonts, CSS)
- Background Sync for any actions taken offline (e.g., marking alert as read, logging a quick note) – sync when connectivity returns.
- Custom “Add to Home Screen” prompt (or rely on browser’s) – must appear only when criteria met and not already installed.
- Splash screen with KidneyCare BD logo.

2. BOTTOM NAVIGATION BAR (PRIMARY MOBILE NAV)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Replace any sidebar or top‑tab navigation with a persistent bottom navigation bar containing these 5 tabs (icons from a clean icon set like Phosphor or Heroicons):

1. **Patients** (👥 icon) – Patient list home  
2. **Alerts** (🔔 icon) – All clinical alerts with unread badge  
3. **Today** (📋 icon) – Today’s scheduled follow‑ups & priority actions  
4. **Tools** (🧰 icon) – Quick access to GFR calculator, prescription pad, teleconsult starter  
5. **Profile** (👤 icon) – Doctor’s own settings, offline sync status, language toggle  

- The bar must be fixed at the bottom, 56–64px height, with active state highlight (primary color).
- On larger screens (tablet/desktop), the same navigation can collapse into a left sidebar automatically – but mobile is the primary target.

3. SCREEN‑BY‑SCREEN REDESIGN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

### 3.1 Patients Tab (Home)
- **Search bar** at top with instant filtering by name/ID.
- **Filter chips**: All | Critical | Stage 3+ | Not logged >7d | My Assigned.
- **Patient cards** in a vertical feed:
  - Patient avatar (initials), name, age/sex.
  - Risk badge (colored dot + label: Critical/High/Moderate/Low).
  - Mini metric: latest eGFR + trend arrow (↑↓→).
  - Last vitals date & time.
- Tap a card → slides into full patient detail.
- Pull‑to‑refresh with skeleton loading.
- Swipe‑left on a card to quickly call or message (if telemedicine active).

### 3.2 Alerts Tab
- **Unread badge** on tab icon showing count.
- Alerts grouped into collapsible sections: CRITICAL, WARNING, REMINDER.
- Each alert card:
  - Alert type icon (red exclamation, yellow triangle, blue info).
  - Patient name, short message (e.g., “eGFR dropped 18% in 30 days”).
  - Timestamp (relative time: “2h ago”).
  - Tap → deep‑links to the relevant patient timeline with the problematic data highlighted.
- Swipe‑to‑dismiss (marks as read) or “Mark all as read” button.
- Works offline: cached alerts shown; read status synced later.

### 3.3 Today Tab
- “Today’s Focus” aggregator:
  - List of patients with scheduled follow‑ups today (based on last visit or system recommendation).
  - Patients with critically missed vitals (overdue >7d).
  - Upcoming teleconsult appointments.
- Tap any item → go to that patient.
- If empty, show a calming “All caught up!” illustration in Bengali.

### 3.4 Tools Tab
- A grid of large, tappable tool cards (2‑column on mobile):
  - **GFR Calculator** (opens full‑screen calculator with 3‑equation comparison).
  - **Issue Prescription** (structured form, live Bengali preview, QR code).
  - **Teleconsult** (quick launch: search patient then start call).
  - **Compare Equations** (side‑by‑side MDRD/CG/CKD‑EPI for a selected patient).
  - **CKD Staging Reference** (pocket chart for quick look).
- Each tool must also work offline where possible (formulas client‑side, prescription form cached).

### 3.5 Profile Tab
- Doctor’s name, hospital, specialty.
- **Language toggle** (EN / বাংলা) – changes entire PWA UI instantly.
- **Offline sync status** indicator (last synced, pending actions count).
- **Logout** button.
- **About & Help** section.

4. PATIENT DETAIL (DEEP‑LINKED FROM ANY TAB)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
When a doctor taps a patient anywhere, a full‑screen patient profile opens (with back button):
- **Sticky header**: Patient name, risk badge, quick actions (Call, Prescription, Log Vitals).
- **Scrollable timeline** with:
  - Interactive multi‑line chart (BP, creatinine, glucose) – pinch‑zoom on mobile.
  - Milestone markers.
  - Below chart, chronological log cards with vital numbers.
- **GFR comparison card**: 3 equations side‑by‑side, latest result highlighted.
- **Alerts related to this patient** shown inline.
- **Bottom‑fixed action bar** on this screen: “Log Vitals”, “Issue Rx”, “Teleconsult”.

5. DESIGN SYSTEM & UI REQUIREMENTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- **Color palette**: primary #1A6B8A, success #2ECC71, warning #F39C12, danger #E74C3C.
- **Typography**: Noto Sans Bengali for BN, system font for EN. Large, readable sizes (body 16px, labels 14px).
- **Touch targets**: minimum 48x48dp.
- **Cards**: rounded corners (12px), subtle shadows, white background.
- **Animations**: smooth slide transitions between views; micro‑interactions on taps.
- **Empty states**: Bengali illustrations + helpful text (e.g., “No alerts – great job!”).
- **Dark mode** is a nice‑to‑have but not required initially.
- Use Tailwind CSS for rapid styling; React for components.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
6. OFFLINE BEHAVIOR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- All fetched patient lists, alerts, and recent profiles must be cached in IndexedDB (via idb or similar) so the doctor can browse them even in a basement ward with no signal.
- The “Log Vitals” action for a patient must work offline: store in IndexedDB, then use Background Sync to push when online.
- Show a persistent thin banner at the top when offline: “You are offline – data may be stale” in Bengali/English.
- When reconnected, show a toast: “Synced – 3 pending actions uploaded”.
